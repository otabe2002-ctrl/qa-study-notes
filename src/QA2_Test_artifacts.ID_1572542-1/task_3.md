# Test Plan -- 21-School / Раздел «Корпоративное обучение»

1.  **Вступление**

Настоящий документ является тест-планом проекта 21-School / / Раздел
«Корпоративное обучение»

Цели тестирование -- убедиться, что пользователи могут корректно
ознакомиться с программами корпоративного обучения и отправить заявку
без ошибок.

Тестирование проводится для веб-версии сайта и охватывает
пользовательские сценарии взаимодействия с контентом и формами страницы

2.  **Функциональности, которые будут протестированы**

В рамках тестирования проверяются следующие функциональности:
- Загрузка страницы и корректное отображение контента
- Информационные блоки о корпоративных программах и интенсивах
Форма заявки на обучение

- Ввод данных пользователем

- Обязательные и необязательные поля

- Валидация введённых данных

- Отправка формы

**3) НЕ будут протестированы**

- Интеграции с CRM и сторонними сервисами обработки заявок

- Другие разделы сайта, не относящиеся к корпоративному обучению

- Нефункциональные виды тестирования (нагрузочное, security)

**4) Тестовые единицы**

Тестирование выполняется по следующим логическим блокам страницы:

-   Header сайта:

    -   отображение логотипа;

    -   работа навигационных ссылок.

-   Основной информационный блок:

    -   корректность текстов;

    -   отображение изображений и иконок.

-   Блоки с описанием программ:

    -   структура карточек;

    -   читаемость и полнота информации.

-   Форма заявки:

    -   ввод корректных данных;

    -   ввод некорректных данных;

    -   отправка пустой формы;

    -   отображение ошибок валидации.

-   Footer сайта:

    -   ссылки;

    -   контактная информация.

**5) Test approaches (Подходы и техники)**

В тестировании используются следующие подходы и техники:

-   Функциональное тестирование пользовательских сценариев.

-   Тестирование форм и валидаций.

-   UI-тестирование (визуальная проверка элементов страницы).

-   Кросс-браузерное тестирование (Chrome, Firefox, Edge, Safari).

-   Проверка адаптивности (desktop, tablet, mobile).

Тестирование проводится вручную.

**6) Критерии приёмки (Pass / Fail)**

**Pass:**

-   Страница корректно загружается.

-   Все элементы интерфейса отображаются корректно.

-   Форма заявки успешно отправляется при корректных данных.

-   Пользователь получает сообщение об успешной отправке заявки.

-   Критических дефектов не обнаружено.

**Fail:**

-   Страница не загружается или загружается с ошибками.

-   Форма заявки не отправляется.

-   Ошибки валидации отсутствуют или работают некорректно.

-   Обнаружены критические дефекты, блокирующие основной сценарий
пользователя

**7) Приостановка/Возобновление**

Приостановка тестирования возможна, если:

-   Страница недоступна.

-   Форма заявки полностью неработоспособна.

Возобновление тестирования выполняется после:

-   Исправления критических дефектов.

-   Подтверждения доступности страницы.

**8) Задачи тестировщиков**

-   Выполнение тестирования согласно данному тест-плану.

-   Фиксация найденных дефектов в баг-трекинговой системе.

-   Приложение скриншотов и описаний дефектов.

-   Проведение ретестов после исправления ошибок.

-   Подготовка краткого отчёта о результатах тестирования.
