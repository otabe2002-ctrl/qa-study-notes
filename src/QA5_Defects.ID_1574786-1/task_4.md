# Сценарий 1

### Разработчик говорит "Not a bug"

Примеры действий QA

1. Зафиксировать ожидаемое требование опираясь:
   - Требований
   - Найденных паттернов в других частях системы
     И аргументировать на основе этого почему это Bug
2. Задать уточняющие вопросы. Обсудить с разработчиком
3. Подключить третью сторону
4. Собрать доказательства

* Скриншоты, видео воспроизведения дефекта
* Логи браузера или сервера
* Сравнение с ожидаемым поведением по ТЗ и аналогичных компонентов

# Сценарий 2

### Менеджер ставит Low priorit для критичного бага

Примеры действий QA

1. Показать последствия, если баг уйдет в релиз, выстроив логическую цепочку пользовательских сценариев
2. Предложить альтернативы
3. Попытаться понять с точки зрения бизнес-плана
4. Предложить альтернативы
