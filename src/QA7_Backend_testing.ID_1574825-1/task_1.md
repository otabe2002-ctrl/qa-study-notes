**REST API** - ахитектурный стиль построения распределенных систем. Используется для создания веб-служб, которые работают по протоколу HTTP.

Основная идея REST заключается в том, что проводится работа с ресурсами (пользователи, товары, заказы) Каждый ресурс имеет свой уникальный идентификатор (URL) А взаимодействие с этими ресусрами происходит с помощью стандартных HTTP-методов.

| Методы HTTP | Описание                                                                                                                                                                                                                                                                                                                                         | Примеры использования                                                                                                                |
| ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| GET               | Используется для получения данных о ресурсе<br />или коллекции ресурсов. Запросы GET не должны имезнять. **READ**                                                                                                                                                 | GET  /users  - получить список пользователей                                                                                |
| POST              | Применяется для создания нового ресурса. Данные для нового ресурса<br />обычно передаются в теле запроса. Меняет состояние (отправка формы, загрузка, <br />создание пользователя). **CREATE/UPDATE** | POST /users - создать нового пользователя<br />(данные нового пользователя в теле запроса)) |
| PUT               | Используется для полного обновления существующего ресурса. Можно отправить новую<br />версию ресурса, которая полностью заменяет старую по известному URL                                                                  | PUT /users/123 - полностью<br />обновить данные пользователя с ID 123                                                |
| DELETE            | Служит для удаления ресурса                                                                                                                                                                                                                                                                                                      | DELETE /users/123                                                                                                                                        |

JSON (JavaScript Object Notation) -  формат для обмена между сервером и клиентом, самый популярный на данный момент.  Представляет данные в виде:

* Объектов: неупорядоченный набор пар "ключ: значение", заключенный в фигурные скобки
* Массивов: упорядоченный список значений , заключенный в квадратные скобки []

Пример JSON:

```
{
"api": "v.2.7"
,"user":  {
	"id" : 1024
	"name": "Jane Doe"
	"roles": ['admin", "editor"]
	"active": true
	}
}
```

**Преимущества JSON перед XML и YAML**

* Меньше избыточности. JSON гораздно более лаконичен. XML требует открывающихся и закрывающихся тегов для каждого элемента, что делает громоздким.
* **XML** : `<user><name>Jane Doe</name><active>true</active></user>`
* **JSON** : `{"user": {"name": "Jane Doe", "active": true}}`
* Скорость парсинга: JSON легче и быстрее рабирать (парсить), что критично для производительности веб-приложений.
* Нативная поддержка в JavaScript: JSON - это родной для JavaScript формат что делает его идеальным выбором для веб-разработки
* Поддержка языков: поддерживается большинством языков программирования

JSON Schema. Инструмент для валидации стрктуры JSON-документов. Позволяет описать ожидаемую структуру данных.  Это шаблон, который определяет, как должен выглядеть прввильный JSON.

Нужна для:

1. Валидации(проверка): убедиться, что полученные данные соответствуют ожтданиям. Например, проверить, что поле "email" действительно содержит email, а не просто набор букв).
2. Документация: описывает структуру данных для других разработчиков.
3. Тестирование: Позволяет автоматически генерировать тестовые данные.
4. Безопасность: Отсеивает некорректные ланные на входе в систему

   Пример. Допустим, есть JSON, описывающий пользователя:
   {
   "name": "Иван Петров",
   "age": 30,
   "isStudent": false
   }

JSON  Schema, которая описывает этот объект, будет выглядеть так:

```
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "name": {
      "type": "string",
      "description": "Полное имя пользователя"
    },
    "age": {
      "type": "integer",
      "description": "Возраст в годах",
      "minimum": 0
    },
    "isStudent": {
      "type": "boolean",
      "description": "Является ли пользователь студентом"
    }
  },
  "required": ["name", "age"]
}
```

Сериализация и десериализация в контексте JSON - это процессы преобразования данных между форматом, понятным программе (объекты в памяти), и форматом для хранения/передачи(JSON-строка)

Сериализация - процесс превращения живого объекта из кода (Наприме, Python-словаря, JavaScript-объекта или Java-класса) в строку JSON, чтобы отправить данные по сети (в REST запросе) или сохранить файл

```
import jsonuser_dict = {
    "name": "Иван",
    "age": 30,
    "isAdmin": False
}Превращаем словарь в строку

JSONjson_string = json.dumps(user_dict)print(json_string)
Вывод: ''(Кириллица превратилась в Unicode, это норма для чистого JSON)
```

Десериализация - Распаковка

Это обратный процесс. Берется строка JSON(которую получили от API или прочитали из файла) и превращается обратано в объект на языке программирования, чтобы было удобно работать

```
// --- Десериализация (JSON строка -> Объект) ---
const jsonFromServer = '{"name":"Иван","age":30,"isAdmin":false}';// Превращаем строку обратно в объект
const userObject = JSON.parse(jsonFromServer);console.log(userObject.name); // Вывод: Иван (уже можно обратиться к полю)
console.log(userObject.age);  // Вывод: 30
```

# Пример запроса и ответа из **Auth0 Authentication API**

#### Пример JSON-ответа: ID Token (токен идентификации)

Когда пользователь успешно проходит аутентификацию через Auth0, сервис возвращает ID Token — это JSON Web Token (JWT), который содержит информацию о пользователе. Вот пример такого токена в декодированном виде (то, что внутри токена после расшифровки)

```
{
  "iss": "https://dev-example.auth0.com/",
  "sub": "auth0|123456",
  "aud": "YOUR_CLIENT_ID",
  "exp": 1311281970,
  "iat": 1311280970,
  "name": "Иван Петров",
  "given_name": "Иван",
  "family_name": "Петров",
  "gender": "male",
  "birthdate": "1990-10-31",
  "email": "ivan.petrov@example.com",
  "email_verified": true,
  "picture": "https://example.com/avatar.jpg"
}
```

#### Таблица 1: Ключи ID Token и их типы данных

| Ключ           | Тип данных | Значение из примера | Описание                                                                                                                                        |
| :----------------- | :------------------ | :----------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `iss`            | string              | `"https://dev-example.auth0.com/"` | **Issuer (издатель)** — URL сервера авторизации, который выпустил токен                            |
| `sub`            | string              | `"auth0\|123456"`                   | **Subject (субъект)** — уникальный идентификатор пользователя                                          |
| `aud`            | string              | `"YOUR_CLIENT_ID"`                 | **Audience (аудитория)** — идентификатор приложения, для которого предназначен токен |
| `exp`            | integer             | `1311281970`                       | **Expiration time** — время истечения токена (Unix timestamp)                                                                |
| `iat`            | integer             | `1311280970`                       | **Issued at** — время выпуска токена (Unix timestamp)                                                                          |
| `name`           | string              | `"Иван Петров"`          | Полное имя пользователя                                                                                                            |
| `given_name`     | string              | `"Иван"`                       | Имя (без фамилии)                                                                                                                          |
| `family_name`    | string              | `"Петров"`                   | Фамилия                                                                                                                                          |
| `gender`         | string              | `"male"`                           | Пол пользователя                                                                                                                         |
| `birthdate`      | string              | `"1990-10-31"`                     | Дата рождения в формате `ГГГГ-ММ-ДД`                                                                                      |
| `email`          | string              | `"ivan.petrov@example.com"`        | Адрес электронной почты                                                                                                            |
| `email_verified` | boolean             | `true`                             | Флаг, подтвержден ли email (`true` — подтвержден, `false` — нет)                                                   |
| `picture`        | string              | `"https://example.com/avatar.jpg"` | URL аватара пользователя                                                                                                             |
