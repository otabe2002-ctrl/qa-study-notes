* **Имя:** `_Secure-ab-group`
* **Тип:** технический / аналитический (A/B-тестирование)
* **Предполагаемое назначение:**

  Используется для A/B-тестов. Сайт делит пользователей на группы, чтобы проверять разные версии интерфейса, кнопок, цен, рекомендаций.

**__Secure-access-token**

* **Имя:** `__Secure-access-token`
* **Тип:** аутентификационный (безопасный токен доступа)
* **Предполагаемое назначение:**

  Это токен авторизации. Он подтверждает, что ты вошёл в аккаунт.

  С его помощью сервер понимает: «Да, это тот самый пользователь, пусть заходит».

**__Secure-ETC**

* **Имя:** `__Secure-ETC`
* **Тип:** технический / служебный
* **Предполагаемое назначение:**

  Судя по названию и формату, это внутренний служебный идентификатор сессии или устройства.Используется для стабильной работы сайта, отслеживания состояния пользователя или защиты от подозрительной активности.Этот идентификатор специально передаётся на другие сайты (в iframe), чтобы обеспечивать работу платёжных виджетов и кнопок авторизации.

**__Secure-ext_xcid**

* **Имя:** `__Secure-ext_xcid`
* **Тип:** маркетинговый / идентификационный
* **Предполагаемое назначение:**

  Вероятно, используется для внешней аналитики или рекламных интеграций.

  Помогает понять, откуда ты пришёл: реклама, партнёрская ссылка, внешний сервис.

**__Secure-refresh-token**

* **Имя:** `__Secure-refresh-token`
* **Тип:** аутентификационный

# Флаги

**Secure** — это атрибут, который указывает, что кука передаётся только по защищённому соединению HTTPS. Если сайт работает через HTTP, такая кука не отправляется, что защищает её от перехвата в открытых сетях, например, в публичном Wi-Fi.

**HttpOnly** — это атрибут, который делает куку недоступной для JavaScript на клиентской стороне. Она может использоваться только сервером через HTTP(S)-запросы. Это помогает защититься от XSS-атак, когда злоумышленник пытается украсть куки через скрипты на странице.

**SameSite** — атрибут, который управляет тем, когда браузер отправляет куки при кросс-доменных запросах, что важно для защиты от CSRF. Значения атрибута бывают три:

* **Strict** означает, что кука отправляется только на том же сайте, который её установил; переходы с других сайтов или запросы через iframe, img или fetch куку не отправляют.
* **Lax** — кука не отправляется при автоматических кросс-доменных запросах, но отправляется при переходах по ссылкам с других сайтов.
* **None** — кука отправляется всегда, даже при кросс-доменных запросах, но для этого нужно использовать атрибут вместе с Secure. Чаще всего используется для кросс-доменной авторизации или интеграции с внешними сервисами.

###### 1.__Secure-ab-group

Флаги: **Secure, SameSite=Lax**

**Secure** флаг - обязателен для безопасности, SameSIte=Lax

Для куки, которая хранит номер группы A/B-теста, разработчики выбрали  **SameSite=Lax** , потому что этот режим решает главную задачу:  **сохранять пользователя в одной и той же группе при любых переходах на сайт** , но при этом не отправлять куку в опасные места.

**Если бы выбрали Strict** — кука не передавалась бы при переходе по ссылке из Telegram, почты или мессенджера. Пользователь, кликнув по ссылке на товар, попал бы на сайт без своей метки, и Ozon показал бы ему случайную версию интерфейса. Это сломало бы чистоту эксперимента.

**Если бы выбрали None** — кука передавалась бы абсолютно везде: во встроенные картинки, iframe, скрипты с других сайтов. Это создаёт лишние риски и не нужно, потому что метка A/B-теста не должна работать на чужих ресурсах.

Флаг HttpOnly не стоит, так как без доступа из JS сайт не узнает, какую версию интерфейса показать (старую или новую). A/B-тест сломается. В куке нет ничего опасного.

**2.__Secure-access-token**

Флаги: **Secure, SameSite=Lax,HttpOnly**

**Secure, HttpOnly** - по причине безопасности. **SameSite = Lax** -  так как при переходе по ссылке пользователь сразу авторизуется на своем аккаунте.

**3.__Secure-ETC**

Флаги: **HttpOnly, SameSite= None, Seruce**

Стоит None для перехода по внешним платежным окнам, не теряя данные пользователя

**4.__Secure-ext_xcid**

Флаги: **HttpOnly, SameSite= None, Seruce**
SameSite= None для сбора полной информации и анализа всех переходов по внешним ссылкам партнеров. Если бы стоял Lax, то данные были бы некорректными.

**5.__Secure-refresh-token**

Флаги: **HttpOnly, SameSite= Lax, Seruce**

Refresh-токену Lax нуженровно для того же , что и access-токену:
чтобы пользователь не вылетал из аккаунта при переходах по ссылкам, и сессия обновлялась автоматически.

**6__Secure-user-id**

Флаги: **HttpOnly, SameSite= Lax, Seruce**

Lax Позволяет сайту сразу узнать тебя при переходе по ссылке, чтобы показать персонализированный интерфейс.
