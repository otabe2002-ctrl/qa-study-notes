# Категории тестирования и их виды

| Категория                                                                         | Виды                                                                                                                                                                                                                                                                                                                                            |
| ------------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **По стадии проведения процесса тестирования** | Unit-тестирование, Интеграционное тестирование, Системное тестирование, Альфа-тестирование, Бета-тестирование, Smoke-тестирование, Регрессионное тестирование, Приёмочное (UAT) тестирование |
| **По знанию устройства системы**                            | Black-box (без знания внутреннего кода), White-box (полное знание кода и архитектуры), Grey-box (частичное знание)                                                                                                                                                               |
| **По направленности тестового сценария**            | Функциональное (проверка функций), Нефункциональное (Performance, Security, Usability, Compatibility, Reliability, Accessibility)                                                                                                                                                                      |
| **По выполнению кода**                                               | Статическое (анализ без выполнения кода), Динамическое (выполнение кода и проверка поведения)                                                                                                                                                                         |

| Сценарий тестирования                                                                                                                                                                                                      | По знанию устройства системы | По направленности тестового сценария                                                                                                         | По выполнению кода                                                                                                  |
| ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| **Unit-тестирование:** проверка функции расчета комиссии за перевод                                                                                                                   | White-box                                             | Функциональное – проверка корректности расчёта комиссии при разных суммах                                    | Динамическое – автоматизированное выполнение модульных тестов               |
| **Alfa-тестирование:** тестирование макета интерфейса силами дизайнеров и продукт-менеджеров на удобство и логику переходов          | Black-box                                             | Нефункциональное (Usability) – проверка удобства, логики переходов, цветов кнопок                                 | Статическое – анализ макета без выполнения кода                                            |
| **Alfa-тестирование:** тестирование модуля перевода средств между счетами пользователя                                                                              | Grey-box                                              | Функциональное – проверка корректности расчета комиссии, обработки ошибок, верификации счета | Динамическое – выполнение операций перевода в изолированной среде         |
| **Интеграционное тестирование:** проверка взаимодействия модуля переводов с модулем уведомлений                                                        | Grey-box                                              | Функциональное – проверка, что при переводе приходит уведомление получателю                                 | Динамическое – проверка связей между модулями                                               |
| **Smoke-тестирование:** проверка критической функциональности перед Beta перед выпуском для бета-тестеров                                                | Black-box                                             | Функциональное – быстрая проверка основных операций (вход, перевод, выход)                                      | Динамическое – быстрые критические тесты                                                        |
| **Beta-тестирование: п**роверка приложения реальными пользователями на различных устройствах и версиях ОС                                           | Black-box                                             | Нефункциональное (Compatibility, Performance) – проверка на разных устройствах, сетях, версиях Android/iOS            | Динамическое – долгосрочное тестирование в реальных условиях                  |
| **Регрессионное тестирование:** проверка, что новая функция "сохранение шаблонов переводов" не сломала существующий функционал | Grey-box                                              | Функциональное – проверка всех ранее работающих операций (обычные переводы, быстрые платежи)   | Динамическое – автоматизированное или ручное выполнение набора тестов |
| **UAT (Приёмочное тестирование):** проверка соответствия требованиям заказчика перед выпуском в Production                                                   | Black-box                                             | Функциональное – полная проверка всех требований спецификации (переводы, комиссии, отчеты)       | Динамическое – полное тестирование всех сценариев пользователя              |
